# Graph Package Blob v1

## Purpose

The Graph Package Blob is the binary format used to describe an executable DSP pipeline.

It contains all information required for the runtime to:

* instantiate DSP modules
* allocate buffers
* configure execution order
* initialize module parameters

The blob is generated by the **graph compiler** and consumed by the **runtime engine**.

---

# 1. Endianness and Primitive Types

All integer values in the blob are **little-endian**.

Primitive types:

| Type  | Size    |
| ----- | ------- |
| `u8`  | 1 byte  |
| `u16` | 2 bytes |
| `u32` | 4 bytes |
| `u64` | 8 bytes |
| `i32` | 4 bytes |
| `f32` | 4 bytes |

Blob structures are **packed (no padding)**.

Runtime structures may be aligned as needed after loading.

---

# 2. File Structure

```
[FileHeader]
[SectionHeader + Payload]
[SectionHeader + Payload]
...
[CRC32]
```

The blob is composed of a fixed header followed by a sequence of sections.

---

# 3. File Header

The file header is **32 bytes**.

| Offset | Size | Field         | Meaning                       |
| ------ | ---- | ------------- | ----------------------------- |
| 0      | 4    | magic         | ASCII `"GRPH"`                |
| 4      | 1    | version_major | = 1                           |
| 5      | 1    | version_minor | = 0                           |
| 6      | 2    | header_bytes  | = 32                          |
| 8      | 4    | file_bytes    | total blob size including CRC |
| 12     | 4    | target_abi    | ABI identifier                |
| 16     | 8    | graph_uuid_lo | UUID low bits                 |
| 24     | 8    | graph_uuid_hi | UUID high bits                |

---

# 4. CRC

The final 4 bytes of the file contain a CRC32.

CRC is computed over:

```
bytes [0 ... file_bytes-4]
```

Polynomial: standard Ethernet CRC32 (same as zlib).

---

# 5. Section Format

Each section consists of a header followed by a payload.

### Section Header (16 bytes)

| Offset | Size | Field      | Meaning                 |
| ------ | ---- | ---------- | ----------------------- |
| 0      | 4    | sect_type  | Section identifier      |
| 4      | 4    | sect_bytes | Payload size            |
| 8      | 4    | sect_flags | Reserved                |
| 12     | 4    | sect_crc32 | Optional CRC of payload |

Payload immediately follows the header.

---

# 6. Loader Rules

Loaders must:

* ignore unknown section types
* skip unknown sections using `sect_bytes`
* require required sections for the version
* validate file CRC before execution

---

# 7. Section Types

```c
enum SectType : uint32_t
{
  SECT_REQUIRES        = 1,
  SECT_HEAPS           = 2,
  SECT_BUFFERS         = 3,
  SECT_NODES           = 4,
  SECT_SCHEDULE        = 5,
  SECT_PARAM_DEFAULTS  = 6,
  SECT_METADATA_MIN    = 7,
  SECT_GRAPH_CONFIG    = 8
};
```

---

# 8. GRAPH_CONFIG Section

Defines global graph execution parameters.

```
u32 sample_rate_hz
u32 block_multiple_N
```

### Definitions

| Field              | Meaning                                            |
| ------------------ | -------------------------------------------------- |
| `sample_rate_hz`   | graph sample rate                                  |
| `block_multiple_N` | number of base blocks accumulated before execution |

---

# 9. Runtime Block Model

The runtime host integration layer defines:

```
B = base block size
```

Example:

```
DMA interrupt size
Audio callback size
```

The graph execution block size is:

```
graph_block_frames = B × N
```

Where:

```
N = block_multiple_N
```

The runtime accumulates `N` base blocks before executing the graph.

---

# 10. REQUIRES Section

Lists modules required by the graph.

```
u32 count

repeat count
  u32 module_id
  u16 module_ver_major
  u16 module_ver_minor
  u32 required_caps
```

---

# 11. HEAPS Section

Defines memory heaps used by the runtime.

```
u32 heap_count

repeat heap_count
  u32 heap_id
  u32 heap_kind
  u32 heap_bytes
  u32 heap_align
```

Example heap kinds:

| Value | Meaning   |
| ----- | --------- |
| 0     | general   |
| 1     | state     |
| 2     | parameter |
| 3     | IO        |

---

# 12. BUFFERS Section

Defines audio buffer storage and alias relationships.

```
u32 buf_count

repeat buf_count
  u32 buf_id
  u8  buf_type
  u8  format
  u16 flags

  u32 heap_id
  u32 offset_bytes

  u32 size_bytes

  u16 slots
  u16 stride_bytes

  u32 base_buf_id

  u16 channels
  u16 frames_per_buffer
```

---

### Buffer Types

| Value | Meaning |
| ----- | ------- |
| 0     | OWNED   |
| 1     | VIEW    |
| 2     | ALIAS   |

---

### Format Enum

```c
enum AudioFormat : uint8_t
{
  FMT_F32 = 1,
  FMT_S16 = 2
};
```

---

### Buffer Fields

| Field               | Meaning                              |
| ------------------- | ------------------------------------ |
| `heap_id`           | heap containing the buffer           |
| `offset_bytes`      | offset within heap                   |
| `size_bytes`        | size of buffer memory                |
| `slots`             | number of buffer slots               |
| `stride_bytes`      | sample stride                        |
| `base_buf_id`       | referenced buffer for views/aliases  |
| `channels`          | channel count                        |
| `frames_per_buffer` | frames processed per graph execution |

Typically:

```
frames_per_buffer = B × N
```

---

# 13. NODES Section

Defines module instances.

```
u32 node_count

repeat node_count
  u32 node_id
  u32 module_id

  u32 state_heap_id
  u32 state_offset

  u32 state_bytes
  u32 state_align

  u32 init_bytes
  u32 param_block_bytes

  [init_blob]
```

---

### Node State Allocation

For Blob v1:

```
state_offset = 0
```

Runtime allocates node state sequentially from the state heap.

This simplifies compiler design.

---

# 14. SCHEDULE Section

Defines execution order.

```
u32 op_count

repeat op_count
  u8  op_type
  u8  n_in
  u8  n_out
  u8  flags

  u32 node_id

  u32 in_buf_ids[n_in]
  u32 out_buf_ids[n_out]
```

---

### Operation Types

```
1 = CALL
```

---

### Execution Semantics

Each schedule entry executes **once per graph block**.

```
graph_block_frames = B × N
```

---

# 15. PARAM_DEFAULTS Section

Stores default parameter values.

```
u32 node_count

repeat node_count
  u32 node_id
  u32 param_bytes
  u8  param_data[param_bytes]
```

The encoding of `param_data` is module specific.

---

# 16. METADATA_MIN Section (Optional)

Optional parameter metadata used by tools.

```
u32 entry_count

repeat entry_count
  u32 node_id
  u32 param_id
  u8  type
  u8  flags
  u16 name_len

  char name[name_len]

  f32 min_f32
  f32 max_f32
```

---

### Metadata Types

| Value | Meaning   |
| ----- | --------- |
| 1     | f32       |
| 2     | i32       |
| 3     | u32       |
| 4     | raw bytes |

---

# 17. Parameter Flags

| Flag | Meaning             |
| ---- | ------------------- |
| bit0 | smoothing supported |

---

# 18. Summary

The Graph Package Blob provides a portable representation of a DSP pipeline.

Key design characteristics:

* deterministic memory layout
* no dynamic graph construction
* runtime-allocated node state
* heap-offset buffers
* block execution defined by `N × B`

This structure allows the runtime to remain stable while pipelines evolve.
